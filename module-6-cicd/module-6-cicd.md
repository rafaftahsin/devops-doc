# CI/CD

আগের মডিউল গুলাতে আমরা যা কিছু ডেপ্লয় করেছি বিল্ড করেছি সব কিছু মেনুয়ালি করেছি। কিন্তু প্রাকটিকাল সিনারিউতে আমরা সব যদি মেনুয়ালি বিল্ড এবং ডেপ্লয় করতে যাই তাহলে অনেক সময়ের অপচয় হবে, তাছাড়া যত বেশী ম্যানুয়াল টাস্ক হয় তত বেশী হিউম্যান এরর হওয়ার সম্ভবনা বেশী থাকে। এই মডিউলে আমরা আমাদের প্রজেক্ট যাতে অটোমেটিকালি বিল্ড হয় সেই ব্যাবস্থা করব। 

প্রথমে আমরা Github Action এর মাধ্যমে আমাদের ব্যাকেন্ড এবং ফ্রন্ট এন্ড এপলিকেশন বিল্ড করব। গিটহাব একশনকে এমনভাবে কনফিগার করব যাতে একটা নির্দিষ্ট ব্রাঞ্চে পুশ হলেই অটোমেটিকালি Github Action ট্রিগার করে বিল্ড আর্টিফেক্ট জেনারেট করে ফেলে। 

তার পর প্রথমে আমরা মেনুয়ালি বিল্ড আরটিফেক্ট গুলিকে প্রডাকশনে রান করব। 

এরপর আমরা Github Action এর সাথে আমাদের সার্ভারকে কানেক্ট করব যাতে আমাদের সার্ভার Github Runner হিসেবে কাজ করে। এবং দেখব কিভাবে এই ডেপ্লয়মেন্টকেও অটোমেট করে ফেলা যায়। 

এই যে কোড পুশ এর সাথে সাথে কোড বিল্ড হয়ে আর্টিফেক্ট তৈরী করতেছে এই প্রসেসকে বলা হয় Continuous Integration.
আর এই যে অটোমেটিক ডেপ্লয় হয়ে যাচ্ছে এই প্রসেস কে বলা হয় Continuous Delivery.
